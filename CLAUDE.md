## Project Overview

A two-phase WSL environment bootstrap system:

1. **Export phase** — Snapshots the current system state. Fetches all installed packages from pacman and AUR, and produces both a manifest file and a generated install script.
2. **Setup phase** — Executes on a fresh WSL instance. Installs all packages and wires up the full environment using configs from a dotfiles repository.

The target environment is:
- Shell: `zsh` (no oh-my-zsh, custom config)
- Editor: `neovim`
- Configs: single dotfiles git repo cloned to `~/dotfiles`; nvim uses `init.lua`

## The Two Phases

### Phase 1 — Export (`export/export.sh`)

**Purpose:** Snapshot the current WSL environment so it can be reproduced elsewhere.

**What it does:**
- Queries `pacman` for all explicitly installed packages
- Queries `yay` for AUR packages
- Writes a `manifest/packages.json` with package names and source (pacman/AUR)
- Generates `scripts/install-packages.sh` — a ready-to-run script that installs everything

**Manifest format (`packages.json`):**
```json
{
  "generated_at": "2026-04-22T10:00:00Z",
  "pacman": ["zsh", "neovim", "git", "..."],
  "aur": ["yay", "..."]
}
```

**Output:** Both files must be committed to the repo after running export.

---

### Phase 2 — Setup (`setup/setup.sh`)

**Purpose:** Bootstrap a clean WSL instance into a fully configured environment.

**What it does:**
1. Installs `yay` (AUR helper) if not present
2. Runs `scripts/install-packages.sh` to install all packages from the manifest
3. Clones the dotfiles repo to `~/dotfiles`
4. Symlinks `~/dotfiles/zsh/.zshrc` → `~/.zshrc`
5. Symlinks `~/dotfiles/zsh/config` → `~/.config/zsh`
6. Clones zsh plugins (fzf-tab, zsh-syntax-highlighting, zsh-autosuggestions) into `~/.config/zsh/plugins/`
7. Symlinks `~/dotfiles/nvim` → `~/.config/nvim`
8. Sets zsh as the default shell via `sudo usermod -s`

**Idempotent:** Running setup more than once must be safe — check before creating symlinks, installing packages, etc.

## Dotfiles

Configs for `zsh` and `nvim` live in a single dotfiles git repo cloned to `~/dotfiles` during Phase 2 setup. The repo URL is provided via the `DOTFILES_REPO` env var.

**zsh:**
- No oh-my-zsh
- `.zshrc` at `dotfiles/zsh/.zshrc`, symlinked to `~/.zshrc`
- Modular config files in `dotfiles/zsh/config/`, symlinked to `~/.config/zsh/`
- Plugins (fzf-tab, zsh-syntax-highlighting, zsh-autosuggestions) cloned at setup time into `~/.config/zsh/plugins/`

**nvim:**
- Config at `~/.config/nvim/` (symlinked from `dotfiles/nvim/`)
- Entry point: `init.lua`; uses `lazy.nvim`

See `docs/feature_003_dotfiles/` for details.

## Code Standards

### Shell Scripts

- All scripts use `#!/usr/bin/env bash` (not `/bin/bash`) unless a feature requires zsh explicitly
- `set -euo pipefail` at the top of every script
- Functions are used for any logic longer than ~5 lines
- Prefer explicit variable quoting: `"$VAR"` not `$VAR`
- Scripts must be **idempotent** — safe to run multiple times
- Log output with a consistent prefix, e.g. `[setup]` or `[export]`
- Use `command -v` to check for dependencies before using them

**Example pattern:**
```bash
#!/usr/bin/env bash
set -euo pipefail

log() { echo "[export] $*"; }

check_deps() {
  for cmd in pacman yay jq; do
    command -v "$cmd" &>/dev/null || { log "Missing dependency: $cmd"; exit 1; }
  done
}
```

### JSON / Manifest

- Pretty-printed with 2-space indentation
- Always include a `generated_at` ISO 8601 timestamp
- Package lists are sorted alphabetically

### Naming Conventions

| Thing | Convention |
|-------|-----------|
| Shell scripts | `kebab-case.sh` |
| Directories | `kebab-case/` |
| Shell functions | `snake_case` |
| Shell variables | `UPPER_SNAKE_CASE` for env/globals, `lower_snake_case` for locals |
| Doc folders | `feature_<id>_<name>` |

### Git

- Commits should reference the phase or component: `export:`, `setup:`, `docs:`, `dotfiles:`
- Manifest files (`packages.json`, `install-packages.sh`) are committed after export runs
- Never commit secrets, tokens, or personal paths
