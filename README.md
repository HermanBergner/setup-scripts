# wsl-setup

A two-phase bootstrap system for reproducing an Arch Linux WSL environment.

**Phase 1 (export)** — snapshots the current system into a package manifest and a generated install script.  
**Phase 2 (setup)** — bootstraps a fresh WSL instance: installs packages, clones dotfiles, and wires up the shell and editor.

---

## Prerequisites

- Arch Linux WSL instance
- `pacman` (comes with Arch)
- `yay` (AUR helper — required for export; installed automatically during setup)
- `jq` (JSON processor — required for export)

---

## Quickstart

### Export (run on your existing machine)

```bash
bash export/export.sh
git add manifest/ scripts/
git commit -m "export: update package manifest"
git push
```

### Setup (run on a fresh WSL instance)

```bash
git clone <this-repo> ~/wsl-setup
cd ~/wsl-setup
export DOTFILES_REPO="https://github.com/youruser/dotfiles"
bash setup/setup.sh
```

Re-login or run `exec zsh` after setup completes.

---

## Repository Layout

| Path | Description |
|------|-------------|
| `export/export.sh` | Phase 1: snapshot installed packages |
| `setup/setup.sh` | Phase 2: bootstrap a fresh environment |
| `manifest/packages.json` | Generated package list (committed after export) |
| `scripts/install-packages.sh` | Generated install script (committed after export) |
| `dotfiles/zsh/.zshrc` | zsh configuration |
| `dotfiles/nvim/init.lua` | neovim configuration |
| `docs/` | Per-feature documentation |

> Do not edit `manifest/packages.json` or `scripts/install-packages.sh` by hand — they are generated by `export/export.sh`.
